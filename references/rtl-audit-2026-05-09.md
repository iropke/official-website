# 아랍어 RTL 사전 검토 (2026-05-09)

> Phase B-5 (아랍어 ar) 본 작업 진입 시 필독.
> 본 문서는 `feat/seo-essentials` 브랜치 시점의 14 개 CSS 모듈 + JSX/SVG 자산을 read-only audit 한 결과.
> 코드 변경 없이 작성되었으며, 본격 RTL 패스 작업 시 본 문서를 starting point 로 사용한다.

## 결론

**`<html dir="rtl">` 만으로는 부족.** 별도 RTL CSS 패스가 필요하다.
다만 구조 (Header / Footer / 12-col grid / flex layouts) 는 RTL 자동 반전이 적용되므로 **레이아웃 재설계는 불필요**하고,
**약 30 줄의 기계적 logical-property 치환 + 약 10 곳의 directional 애니메이션·아이콘 분기**로 해결 가능하다.

## High 위험 5종 — 상세

### 1. SubCarousel prev/next nav

**파일**: `src/components/home/SubCarousel/SubCarousel.module.css:151,155,201,205` + `SubCarousel.tsx:82-87, 177, 193`

| 문제 | RTL 영향 |
|---|---|
| `--prev` 버튼 `left:` 고정 | 시각적 next 가 좌측에 위치해야 하는데 prev 가 좌측에 있음 |
| `--next` 버튼 `right:` 고정 | 동일 |
| SVG 좌향 화살표 `M11 3 5 9l6 6` (prev) | 화살표 자체 방향이 LTR 기준 |
| SVG 우향 화살표 `M7 3l6 6-6 6` (next) | 동일 |
| 키보드 핸들러 `ArrowRight → next, ArrowLeft → prev` | RTL 에서 시각적 next 는 ArrowLeft 가 자연 |

**필요 분기**: 버튼 위치 (logical), SVG transform, 키보드 핸들러 모두 `dir` 의존.

### 2. SearchToggle 펼침 애니메이션

**파일**: `src/components/layout/SearchToggle/SearchToggle.module.css:34, 63, 73`

| 문제 | RTL 영향 |
|---|---|
| `.iconButton { right: 0 }` | 아이콘이 시각적 우측에 고정 |
| `.searchForm { left: 0 }` | 폼이 좌측에서 펼쳐짐 |
| `transform: translateX(-10px) → 0` | 좌측 슬라이드 |

**필요 분기**: `inset-inline-end: 0` (아이콘) / `inset-inline-start: 0` (폼) / `translateX` 부호 RTL 에서 반전.

### 3. Header 모바일 패널

**파일**: `src/components/layout/Header/Header.module.css:295, 313, 378, 431`

| 문제 | RTL 영향 |
|---|---|
| 패널 `transform: translateX(20px) → 0` | 우측 슬라이드인. RTL 은 좌측에서 진입해야 자연 |
| 활성 메뉴 항목 `border-left: 3px solid #555` | 표시자 좌측 — RTL 에서 우측이어야 함 |
| 모바일 메뉴 `border-left: none` 재정의 | 동일 |

**필요 분기**: `:dir(rtl)` 전용 transform 부호 + `border-inline-start`.

### 4. FloatingActions 호버 라벨

**파일**: `src/components/layout/FloatingActions/FloatingActions.module.css:8, 71`

| 문제 | RTL 영향 |
|---|---|
| 버튼 자체 `right: 20px` (화면 우측 고정) | 유지 가능 (화면 모서리 고정은 dir 무관) |
| 라벨 `right: calc(100% + 12px)` | 라벨이 버튼 좌측에 표시됨 → RTL 에서는 화면 밖으로 잘림 |

**필요 분기**: 라벨 위치만 `inset-inline-end: calc(100% + 12px)` 로.

### 5. ProjectInquiry 검증 마크 + 체크 글리프

**파일**: `src/app/(frontend)/[locale]/project-inquiry/ProjectInquiry.module.css:39, 214, 370-372, 612`

| 문제 | RTL 영향 |
|---|---|
| `.fieldStatus { right: 18px }` (input trailing edge 체크마크) | RTL 에서 `inset-inline-end: 18px` 가 자연 |
| 체크 글리프 `border-left + border-bottom + transform: rotate(-45deg)` | 모양 반전 필요 |
| `.formCard { margin-right: 20px }` (form ↔ aside 간격) | RTL 에서 margin 방향 반전 |

**필요 분기**: logical property 치환 + 체크 글리프 transform.

## Quick Wins (기계적 치환)

다음은 의미는 그대로이고 logical property 로 바꾸기만 하면 자동 RTL 호환되는 항목들.

| 변경 전 | 변경 후 | 위치 |
|---|---|---|
| `padding-left/right: var(--page-shell-gutter)` | `padding-inline: var(--page-shell-gutter)` | layout-page, Header, Footer 등 5 파일 |
| `margin-left: 0` | `margin-inline-start: 0` | Footer.module.css:53 |
| `margin-left: 6px` | `margin-inline-start: 6px` | LanguageSelector.module.css:17 |
| `margin-right: 20px / 0` | `margin-inline-end: 20px / 0` | ProjectInquiry.module.css:39, 612 |
| `padding-left/right`, `border-left` (Header) | `padding-inline`, `border-inline-start` | Header.module.css:21-22, 378, 431 |
| `left: 16px` (badge), `border-left: 3px` | `inset-inline-start: 16px`, `border-inline-start: 3px` | MegaMenu.module.css:78, 183 |
| `left: 22px` (번호), `right: 24px` (아이콘) | `inset-inline-start: 22px`, `inset-inline-end: 24px` | AIOSGrid.module.css:153, 163, 173 |
| `right: 18px` (체크), `border-left` | `inset-inline-end: 18px`, `border-inline-start` | ProjectInquiry.module.css:214, 372 |

`margin-left: auto; margin-right: auto;` (4 파일에서 centering 용도) 는 **그대로 두면 됨** — 대칭이라 RTL 자동 호환.

## 안전 항목 (변경 불필요)

| 항목 | 사유 |
|---|---|
| 12-column grid (Header), flex layouts (Footer/Insights/AIOSGrid) | `gap`, `space-between`, `flex-end` 는 RTL 자동 반전 |
| `transform: translate(-50%, -16px)` (MegaMenu 센터링) | 좌우 대칭이라 dir 무관 |
| 장식 orb (`Hero.orbA/orbB`, `CtaBanner.banner__orb`) | 시각적 장식이라 좌우 반전이 의미적 차이 없음 — 그대로 두어도 무방 |
| `text-align: center` (CoreMessage) | 대칭 |
| `Insights.module.css`, `CoreMessage.module.css` | 0 개 physical 속성 — 그대로 통과 |

## 권장 진행 순서 (Phase B-5)
